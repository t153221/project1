## 1. Business Purpose

세상에서는 사람의 욕구를 채우기 위한 여러 활동이 존재한다. 그 중에서도 게임 산업은 영화, 책 등의 이야기를 보는 것만으로 욕구를 채워주는 어떤 것이 될 수 있고, 농구, 축구처럼 본인이 플레이함으로서 박진감, 희열 등을 선사해줄 수도 있다. 이렇게 게임 하나로 다양한 욕구를 충족할 수 있다고 생각한다.

본 프로젝트의 목적은 언리얼 엔진을 활용하여 기존의 시각 중심 게임과 차별화된 "청각 기반 공포 게임"을 개발하는 데 있다. 일반적인 공포 게임이 그래픽과 시각적 연출에 의존하는 것과 달리, 본 게임은 시야를 제한하고 공간 음향(Spatial Audio)을 핵심 요소로 활용하여 플레이어가 소리를 통해 환경을 인지하도록 설계한다.

이를 통해 플레이어는 제한된 정보 속에서 방향을 판단하고 위험을 회피해야 하며, 이는 기존 게임과는 다른 몰입형 사용자 경험을 제공한다. 특히 사운드 중심 인터랙션은 비교적 낮은 개발 비용으로도 강한 긴장감과 공포를 유도할 수 있어, 인디 게임 시장에서 경쟁력을 가질 수 있는 요소로 작용한다.

또한 본 프로젝트는 언리얼 엔진의 오디오 시스템 및 실시간 인터랙션 기능을 활용하여 게임 개발 기술 역량을 향상시키는 데 목적이 있으며, 향후 다양한 플랫폼으로 확장 가능한 콘텐츠 개발의 기반을 마련하는 것을 목표로 한다.

## 3. Use Case List

1) 게임 시작
- Actor: player
- Description: 플레이어는 메인 메뉴에서 게임을 시작하고 게임 환경에 진입한다.

2) 환경 탐색
- Actor: player
- Description: 플레이어는 제한된 시야 속에서 어두운 환경을 탐색한다.

3) 소리 감지
- Actor: player
- Description: 플레이어는 환경음을 통해 탈출 지점의 방향과 거리를 판단한다.

4) 이동 조작
- Actor: player
- Description: 플레이어는 입력 장치를 이용하여 캐릭터를 이동시키고 탐색하거나 위험을 회피한다.

5) 적 회피
- Actor: player
- Description: 플레이어는 적의 추적을 피하며 이동한다.

6) 적 반응
- Actor: system
- Description: 시스템은 플레이어의 위치를 감지하여 적의 이동을 유도한다.

7) 게임 오버
- Actor: system
- Description: 플레이어가 적과 접촉하면 시스템이 게임을 종료한다.

8) 목표 달성
- Actor: player
- Description: 플레이어는 탈출 지점에 도달하여 게임을 클리어한다.

9) 게임 종료
- Actor: system
- Description: 게임 오버 또는 클리어 조건이 충족되면 시스템이 결과 화면을 표시한다.

## 4. Concept of Operation

#### 1) Explore Environment

| 항목 | 내용 |
| -------- | --------------------------------------------------------------------------- |
| Purpose | 플레이어가 제한된 시야 환경에서 게임 공간을 탐색하도록 한다. |
| Approach | 플레이어는 입력 장치를 이용하여 캐릭터를 이동시키며 어두운 공간을 탐색한다. 시야는 제한되어 있으며 주변 정보를 직접 확인하기 어렵다. |
| Dynamics | 플레이어가 이동할 때마다 위치가 변경되며, 이동 과정에서 발소리가 발생한다. |
| Goals | 시야 제한 환경에서의 탐색 메커니즘을 구현한다. |

#### 2) Detect Sound

| 항목 | 내용 |
| -------- | ------------------------------------------------------ |
| Purpose | 플레이어가 소리를 통해 탈출 지점의 위치를 파악하도록 한다. |
| Approach | 탈출 지점에서 발생하는 환경음을 Spatial Audio로 제공하여 방향과 거리를 판단할 수 있도록 한다. |
| Dynamics | 소리는 거리와 방향에 따라 다르게 들리며, 플레이어는 이를 기반으로 이동 방향을 결정한다. |
| Goals | 청각 기반 정보 인지 시스템을 구현한다. |

#### 3) Enemy Detection & Tracking

| 항목 | 내용 |
| -------- | ------------------------------------------------- |
| Purpose | 적이 플레이어를 추적하도록 한다. |
| Approach | 시스템은 AI MoveTo를 이용하여 적이 항상 플레이어 위치로 이동하도록 한다. |
| Dynamics | 플레이어가 움직이면 적이 NavMesh를 통해 경로를 계산하며 지속적으로 추적한다. |
| Goals | 플레이어 추적 AI 시스템을 구현한다. |

#### 4) Game Over & Escape Objective

| 항목 | 내용 |
| -------- | ------------------------------------ |
| Purpose | 게임 실패 및 성공 조건을 제공한다. |
| Approach | 적과 접촉 시 즉시 게임 오버 화면을 표시하고, 탈출 지점 도달 시 클리어 화면을 표시한다. |
| Dynamics | 적을 회피하면서 탈출 지점에 도달하면 게임이 종료된다. |
| Goals | 게임의 명확한 성공 및 실패 조건을 구현한다. |

## 5. Problem Statement

본 프로젝트는 시각적 정보가 제한된 환경에서 플레이어가 청각 정보에 의존하여 게임을 진행하는 공포 게임을 구현하는 것을 목표로 한다. 그러나 이러한 구조는 일반적인 시각 중심 게임과 달리 다음과 같은 문제점을 수반한다.

첫째, 소리 기반 인터랙션의 정확한 구현이 필요하다. 플레이어가 소리를 통해 방향과 거리를 인지할 수 있도록 하기 위해서는 공간 음향(Spatial Audio)의 정밀한 설정이 요구되며, 거리 감쇠(Attenuation)와 방향성(Spatialization)을 고려해야 하는 기술적 어려움이 존재한다.

둘째, 시야 제한 환경에서의 사용자 경험 설계가 어렵다. 화면 정보가 부족할 경우 플레이어는 방향 감각을 잃거나 게임 진행에 어려움을 겪을 수 있으므로, 최소한의 정보 제공과 적절한 피드백 설계가 필요하다.

셋째, 적 인공지능(AI)의 구현이 중요한 문제로 작용한다. 적이 플레이어를 자연스럽게 추적하도록 하기 위해서는 NavMesh 기반 경로 탐색이 필요하며, 비현실적인 움직임을 방지해야 한다.

넷째, 본 프로젝트는 언리얼 엔진의 블루프린트(Blueprint)를 활용하여 구현되기 때문에, 시각적 스크립팅 환경에서 복잡한 로직을 구성하는 데 어려움이 존재한다. 특히 이벤트 처리, 상태 관리, 시스템 간 상호작용을 블루프린트로 설계할 경우 구조가 복잡해질 수 있으며, 디버깅과 유지보수가 어려워질 수 있다.

다섯째, C++를 사용한 직접적인 코드 구현과 비교했을 때, 블루프린트는 성능 최적화와 세밀한 제어 측면에서 제한이 존재한다. 복잡한 연산이나 고성능 처리가 요구되는 경우 C++이 더 적합하지만, 본 프로젝트에서는 개발 효율성과 구현 속도를 고려하여 블루프린트를 중심으로 개발을 진행하였다.

따라서 본 프로젝트는 청각 기반 인터랙션, 제한된 시야 환경 설계, AI 추적 시스템, 블루프린트 기반 로직 구현, 그리고 C++과의 기술적 차이를 고려한 설계라는 문제를 해결하는 것을 주요 과제로 한다.

### Non-Functional Requirements (NFRs)

1. **Performance (성능)**
게임은 원활한 플레이를 위해 안정적인 프레임 레이트(최소 30 FPS 이상)를 유지해야 한다.

2. **Usability (사용성)**
플레이어는 복잡한 조작 없이 직관적인 입력만으로 게임을 진행할 수 있어야 한다.

3. **Audio Quality (오디오 품질)**
방향성과 거리감을 명확하게 전달할 수 있도록 고품질의 공간 음향을 제공해야 한다.

4. **Reliability (신뢰성)**
게임 실행 중 오류나 비정상 종료 없이 안정적으로 동작해야 한다.

5. **Immersion (몰입도)**
사운드와 환경 연출을 통해 플레이어가 높은 수준의 몰입감을 느낄 수 있어야 한다.

6. **Scalability (확장성)**
향후 새로운 맵, 사운드, 기능 등을 추가할 수 있도록 구조를 설계해야 한다.

7. **Responsiveness (반응성)**
플레이어 입력에 대해 즉각적인 반응이 이루어져야 하며, 지연 없이 캐릭터가 움직여야 한다.

## 6. Glossary

**Unreal Engine**
고품질 그래픽과 실시간 렌더링을 지원하는 게임 엔진으로, 본 프로젝트에서 게임을 개발하기 위한 핵심 도구이다.

**Blueprint**
코드를 작성하지 않고 노드 기반의 시각적 방식으로 게임 로직을 구현할 수 있는 언리얼 엔진의 기능이다.

**C++**
언리얼 엔진에서 고성능 로직 구현과 세밀한 제어를 위해 사용되는 프로그래밍 언어이다.

**Object**
게임 내에 존재하는 모든 개체를 의미하며, 캐릭터, 적, 아이템 등과 같이 속성과 기능을 가진 요소를 포함한다.

**Audio System**
게임 내에서 사운드의 재생, 처리 및 제어를 담당하는 시스템으로, 환경음과 효과음을 포함한다.

**Spatial Audio**
소리의 방향과 거리 정보를 반영하여 입체적으로 들리도록 하는 기술로, 플레이어가 청각을 통해 위치를 인지할 수 있도록 한다.

**Attenuation**
소리가 발생한 위치로부터 거리가 멀어질수록 음량이 점차 감소하는 효과를 의미한다.

**Sound Cue**
여러 개의 사운드를 조합하고 랜덤 재생, 반복 등의 설정을 할 수 있는 언리얼 엔진의 오디오 구성 요소이다.

**Rendering**
그래픽 데이터를 기반으로 최종 화면 이미지를 생성하는 과정으로, 조명과 텍스처 등이 포함된다.

**Post Processing**
렌더링된 화면에 추가적인 시각 효과를 적용하는 과정으로, 밝기 조정, 색상 보정, 블러 효과 등을 포함한다.
